    Section 10.2 Selecting the Particular Solution Form

    In the previous section, we saw that the general solution to a linear nonhomogeneous constant coefficient (LNCC) equation has two parts: the homogeneous solution \(y_h\) and the particular solution \(y_p\text{.}\) The homogeneous solution is found using the characteristic equation, while the particular solution is constructed by (1) selecting a form similar to the forcing function, \(f(x)\text{,}\) and (2) determining the coefficients of that form.
    üîó
    In this section, we focus on selecting a form for \(y_p\text{.}\) The key idea is to match the structural form of the forcing function \(f(x)\text{,}\) so that when it is substituted into the differential equation, all the resulting terms simplify to \(f(x)\text{.}\)
    üîó

    Selecting Basic Forms.

    The first step in choosing \(y_p\) is recognizing the structure of \(f(x)\text{.}\) Since \(y_p\) and its derivatives must eventually simplify to match \(f(x)\text{,}\) \(y_p\) should have a similar form. If \(f(x)\) is a polynomial, exponential, sine, or cosine function, then \(y_p\) should be too.
    üîó
    Table¬†150 provides a guide for selecting an initial form of \(y_p\) based on the type of \(f(x)\text{.}\) In each case, \(A, B, C, \dots\) represent the undetermined coefficients we‚Äôll solve for later.
    üîó
    One important note: even if a term is missing in \(f(x)\text{,}\) you should still include it in \(y_p\) if it‚Äôs part of the general form. For example:
    üîó
    üî∏
    üîó
    If \(f(x) = x^2 + 1\text{,}\) the \(x\) term is missing, but \(y_p\) must still include it:
    \begin{equation*} y_p = Ax^2 + Bx + C \text{.} \end{equation*}
    üîó
    üî∏
    üîó
    If \(f(x) = \cos(3x)\text{,}\) then \(y_p = A\cos(3x) + B\sin(3x)\text{.}\) Even though \(\sin(3x)\) isn‚Äôt in \(f(x)\text{,}\) it‚Äôs needed because its derivative produces cosine terms.
    üîó

    When Guessing \(y_p\) Works (and When It Doesn‚Äôt).

    This method of guessing the particular solution based on the forcing function is very effective. However, it does have its limitations. In particular, it only applies when \(f(x)\) can be written as a sum or product of:
    üîó
    • Polynomials, like \(f(x) = 3x^2 - 4x + 5\)
      üîó
      üîó
    • Exponentials, like \(f(x) = 6 e^{2x}\)
      üîó
      üîó
    • Sine and cosine functions, like \(f(x) = \cos(3x) + 2\sin(3x)\)
      üîó
      üîó
    • Any sum or product of these function types
      üîó
      üîó
    Functions such as \(\ln x\text{,}\) \(\tan x\text{,}\) or \(1/x\) fall outside this category. For those, we‚Äôll need a different method entirely. But for the allowed types, this method offers a fast and reliable way to select \(y_p\text{.}\)
    üîó

    Checkpoint 153. üìñ‚ùì When to Guess \(y_p\).

    Combining Basic Forms via Addition.

    If \(f(x)\) is a sum of different function types, say a polynomial plus an exponential, then \(y_p\) should be a sum of the corresponding matching forms.
    üîó

    üåå Example 154. Initial form of \(y_p\).

    Use Table¬†150 to identify the appropriate form of \(y_p\) in each case.
    üîó
    \(\ds y'' + 2y' + 3y = 2e^{w} - 1\)
    üîó
    Solution.
    This matches is exponential plus constant, so:
    \begin{equation*} y_p = Ae^{w} + B \text{.} \end{equation*}
    üîó
    üîó
    üîó
    \(\ds u'' - u = 3\sin(2\theta) + \cos(2\theta)\)
    üîó
    Solution.
    Both terms have the same argument, \(2\theta\text{,}\) the form is the standard row 6
    \begin{equation*} y_p = A\sin(2\theta) + B\cos(2\theta) \text{.} \end{equation*}

    üîó
    üîó
    \(\ds \omega''' + \omega' = 5e^{3w} - 2\sin(4w)\)
    üîó
    Solution.
    This combines exponential and sine, so we need:
    \begin{equation*} y_p = Ae^{3w} + B\sin(4w) + C\cos(4w) \text{.} \end{equation*}

    Combining Basic Forms via Multiplication.

    If the forcing function \(f(x)\) is a product of basic function types‚Äîsuch as a polynomial times an exponential‚Äîthen the particular solution \(y_p\) must also be a product of their matching forms. For example, the differential equation
    üîó
    \begin{equation*} y'' + y' + 5y = 10x^2 e^{6x}\text{,} \end{equation*}
    has a forcing function that is a quadratic times an exponential, so we start with:
    \begin{equation*} y_p = (Ax^2 + Bx + C)(D e^{6x}) \text{.} \end{equation*}
    üîó
    However, be careful: multiplying by \(D\) just makes new coefficients:
    \begin{equation*} y_p = (AD x^2 + BD x + CD) e^{6x} \end{equation*}
    üîó
    Since each pair can be merged as single coefficient, we just relabel them:
    \begin{equation*} y_p = (Ax^2 + Bx + C) e^{6x} \end{equation*}
    üîó
    We‚Äôll use this simplification trick often. It keeps the form of \(y_p\) clean and will make solving for \(A\text{,}\) \(B\) and \(C\) much easier later.
    üîó

    Checkpoint 156. üìñ‚ùì Product of particular solution forms.

    Select the initial form of the particular solution for an LNCC equation with forcing function:
    \begin{equation*} f(x) = x^2 e^{x}\text{.} \end{equation*}
    üîó
    • \(\ds\quad y_p = (Ax^2 + Bx + C)e^{x}\)
    • Correct! The forcing function is the product of a 2nd degree polynomial and an exponential function, so the particular solution should be the product the most general 2nd degree polynomial and an exponential function.
    • \(\ds\quad y_p = Ax^2e^{x}\)
    • Incorrect. Although this form perfectly matches the form of the forcing function, the solution could potentially include \(x\) and free terms.
    • \(\ds\quad y_p = (x^2 + Ax + B)e^x\)
    • Incorrect. The \(x^2\) term also needs a coefficient.
    • \(\ds\quad y_p = (Ax^2 + Bx + C)(De^{x})\)
    • Incorrect. The extra coefficient \(D\) is unnecessary, as it can be absorbed into the constants \(A, B,\) and \(C\text{.}\)
    üîó

    üåå Example 157. Initial form of \(y_p\) - Products of Forms.

    For each of the following forcing functions, select an appropriate initial form of \(y_p\text{.}\) Simplify by combining or eliminating any redundant constants.
    üîó
    \(\ds f(x) = e^{-x}\cos(x)\)
    üîó
    Solution.
    This is an exponential times a cosine:
    \begin{equation*} y_p = Ae^{-x}(B\sin x + C\cos x)\text{.} \end{equation*}
    However, distributing \(A\) shows that it is redundant and we relabel as:
    \begin{equation*} y_p = e^{-x}(A\sin x + B\cos x)\text{.} \end{equation*}
    üîó
    üîó
    üîó
    \(\ds f(x) = e^{6x}e^{-2x}\)
    üîó
    Solution.
    The forcing function simplifies: \(e^{6x}e^{-2x} = e^{4x}\text{,}\) so:
    \begin{equation*} y_p = Ae^{4x} \end{equation*}
    üîó
    üîó
    üîó
    \(\ds f(t) = t^{3}\cos t\)
    üîó
    Solution.
    This is a 3rd degree polynomial times a cosine. So, initially we set:
    \begin{equation*} y_p = (At^3 + Bt^2 + Ct + D)(E\sin t + F\cos t)\text{.} \end{equation*}
    üîó
    Multiplying the polynomial onto sine and cosine, gives us
    \begin{align*} y_p = (At^3 \amp + Bt^2 + Ct + D)E\sin t \\ \amp + (At^3 + Bt^2 + Ct + D)F\cos t \text{.} \end{align*}
    üîó
    Distributing \(E\) and \(F\) shows that they are redundant, so we relabel as follows:
    \begin{align*} y_p = (At^3 \amp + Bt^2 + Ct + D)\sin t \\ \amp + (Et^3 + Ft^2 + Gt + H)\cos t \end{align*}
    üîó
    üîó
    üîó
    \(\ds f(x) = x^2\cos x + x \sin x\)
    üîó
    Solution.
    Here, both terms involve powers of \(x\) times trig functions. But we don‚Äôt need separate guesses‚Äîone form covers both:
    \begin{align*} y_p = (Ax^2 + Bx \amp + C)\cos x \\ \amp + (Dx^2 + Ex + F)\sin x \end{align*}
    üîó
    This form includes the \(x^2\cos x\) term and also handles \(x\sin x\) automatically.
    üîó
    üîó
    üîó
    üîó
    With these examples and the table as a guide, you‚Äôre now equipped to handle both additive and multiplicative combinations in \(f(x)\text{.}\) Next, we‚Äôll deal with one last wrinkle: what to do if \(y_p\) accidentally overlaps with the homogeneous solution \(y_h\text{.}\)
    üîó

    Modifying Particular Solutions.

    We now know how to choose a form for \(y_p\) based on the structure of \(f(x)\text{,}\) but there‚Äôs one final complication: If a term in \(y_p\) a term in the homogeneous solution \(y_h\text{,}\) then that term cannot contribute to generating \(f(x)\text{.}\) Instead, it will give zero, since that is what terms in \(y_h\) do when substituted into the differential equation.
    üîó
    To fix this, we multiply the overlapping part of \(y_p\) by \(x\text{.}\) If there‚Äôs still overlap, we multiply by \(x\) again‚Äîrepeating until \(y_p\) shares no like-terms with \(y_h\text{.}\)
    üîó
    For example, consider:
    \begin{equation*} y'' - 4y' + 3y = e^{3x} \end{equation*}
    The homogeneous solution and the initial particular solution are:
    \begin{equation*} y_h = c_1 e^x + {\DLO c_2 e^{3x}}, \quad y_p = {\DLO A e^{3x}}\text{.} \end{equation*}
    üîó
    Notice that \(y_h\) and in \(y_p\) contain the like terms: \(c_2 e^{3x}\) and \(A e^{3x}\text{.}\) This means that plugging \(y_p\) into the left side of the equation will simplify to \(0\text{,}\) not \(e^{3x}\text{,}\) as required.
    üîó
    So we modify \(y_p\) by multiplying the conflicting term by \(x\text{:}\)
    \begin{equation*} y_p = A x e^{3x} \end{equation*}
    üîó
    Now, \(y_p\) has no terms in common with \(y_h\text{,}\) ensuring that they are independent.
    üîó

    Modification Rule for the Particular Solution.

    If any term in your guess for \(y_p\) also appears in \(y_h\text{,}\) multiply that term by the independent variable \(x\text{.}\) Repeat as needed until all overlapping terms are eliminated.
    üîó
    üîó

    Checkpoint 159. üìñ‚ùì Handling Like Terms.

    What should you do if \(y_p\) contains a term also present in \(y_h\text{?}\)
    üîó
    • Multiply the term in \(y_p\) by the independent variable.
    • Delete the term from \(y_p\text{.}\)
    • Multiply the term in \(y_h\) by the independent variable.
    • Use integration to eliminate the term.
    üîó
